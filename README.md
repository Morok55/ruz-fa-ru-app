## FinAcademia

Веб‑приложение расписания Финансового университета, интегрированное в Telegram Mini App.

- **Telegram Mini App**: `https://t.me/FinAcademiaBot`
- **Назначение**: быстрый и удобный просмотр расписания пар по всему Финансовому университету для студентов и преподавателей.
- **Особенности**: лаконичный тёмный UI, плавные анимации (горизонтальный свайп по дням/неделям, bottom‑sheet с деталями занятий), оптимизированный бэкенд‑прокси и кэширование под высокий трафик. Приложение уверенно работает даже на слабом хостинге, снижая расходы на инфраструктуру.

## 📝 Описание

**FinAcademia** — это SPA на React, работающее как обычный веб‑сайт и как Telegram Mini App через [`Telegram WebApp`](https://core.telegram.org/bots/webapps).  
Фронтенд обращается к лёгкому Python/Flask‑бэкенду, который:

- проксирует официальный API расписания `ruz.fa.ru`,
- кэширует ответы в памяти,
- отдаёт фронтенд‑сборку как статические файлы.

Для пользователя это выглядит как нативное мобильное приложение внутри Telegram с акцентом на:

- **минимальное время до первого экрана расписания**,  
- **комфортное листание недель и дней**,  
- **понятное отображение текущей пары и оставшегося времени**.

Ссылку на живое приложение: [`FinAcademia Mini App`](https://t.me/FinAcademiaBot).

## 🧰 Стек и технологии

**Фронтенд**

- **React 18** (`react`, `react-dom`)
- **Vite 5** (`vite`, `@vitejs/plugin-react`)
- **UI и анимации**
  - `swiper` — горизонтальный свайп по дням и неделям
  - `react-icons` — иконки (поиск, звёзды избранного, копирование почты и т.д.)
- **CSS**
  - кастомная тёмная тема в `frontend/src/css/main.css`
  - мобильный layout под Telegram WebApp (100% высоты, жесты, overscroll‑контроль)

**Бэкенд**

- **Python 3 / Flask 3**
  - `flask`
  - `flask-cors`
  - `Flask-Compress` — сжатие ответов
- **HTTP‑клиент**
  - `requests` с пулом соединений и retry‑логикой
- **Кэширование**
  - `cachetools.TTLCache` — in‑memory кэш ответов расписания
- **Конфигурация**
  - `python-dotenv` — загрузка `.env`
- **WSGI / продакшн**
  - `gunicorn`

**DevOps / прочее**

- `render.yaml` — конфигурация деплоя на Render (сборка фронта и запуск `gunicorn backend.app:app`)
- Git + `.gitignore` (игнорирует `frontend/node_modules`, `frontend/dist`, кэш Python и тестовые артефакты)

## 💻 Установка и запуск

### Локальная разработка

#### 1. Клонирование

```bash
git clone <repo-url>
cd ruz-fa-ru-app
```

#### 2. Бэкенд (Flask API)

```bash
cd backend

# (опционально) виртуальное окружение
python -m venv .venv
# Windows PowerShell:
.venv\Scripts\Activate.ps1

pip install -r requirements.txt

# создать .env из примера и при необходимости поправить значения
copy .env.example .env

# запуск dev-сервера
python app.py
```

По умолчанию сервер стартует на `http://0.0.0.0:8000` и экспонирует:

- `GET /api/health` — health‑check
- `GET /api/groups?term=...` — подсказки групп из локального индекса
- `GET /api/search?term=...` — прокси к `/api/search` RUZ
- `GET /api/schedule/group/<group_id>?start=YYYY.MM.DD&finish=YYYY.MM.DD&lng=1` — прокси расписания

#### 3. Фронтенд (Vite + React)

В отдельном терминале:

```bash
cd frontend
npm install

# режим разработки (с поддержкой HMR)
npm run dev
```

По умолчанию Vite поднимается на `http://localhost:5173`.  
Для разработки используется `frontend/.env.development`:

```env
VITE_API_BASE=http://localhost:8000/api
```

Фронтенд будет ходить в локальный Flask‑бэкенд.

### Продакшн–сборка

#### Ручной билд

```bash
cd frontend
npm run build  # результат в frontend/dist
```

После билда Flask начинает раздавать статику из `frontend/dist`:

- в `backend/app.py`:
  - `app.static_folder = ../frontend/dist`
  - маршруты `/` и `/<path>` отдают `index.html` (SPA‑роутинг).

В продакшене рекомендуется, чтобы `VITE_API_BASE` указывал на относительный `/api`:

```env
# frontend/.env.production
VITE_API_BASE=/api
```

### Docker

_(не указано)_ — в репозитории нет готового `Dockerfile` или `docker-compose.yml`.  
Приложение можно контейнеризировать стандартным образом (слой Python с Flask + слой Node для сборки фронтенда), опираясь на команды из раздела продакшн‑сборки.

## ✨ Возможности

- **Телеграм‑интеграция**
  - Инициализация через `telegram-web-app.js` в `index.html`
  - Отключение вертикальных свайпов и разворачивание WebApp на весь экран
  - Использование `Telegram.WebApp.BackButton` в `GroupSearch` для естественной навигации

- **Поиск и избранные группы**
  - Полноэкранное окно поиска групп с плавным слайд‑эффектом
  - Подсказки групп из локального индекса (`/api/groups`)
  - Список **избранных** групп с хранением в `localStorage` и быстрым выбором

- **Удобная навигация по расписанию**
  - Лента дней недели (`WeekStrip`) с анимацией "плавающей таблетки"
  - Свайп по дням (`Sections`) с предзагрузкой соседних дней и недель
  - Плавные переходы между неделями, в том числе через край (вс → пн и пн → вс)

- **Оптимизация и кэширование**
  - На фронтенде:
    - кэш расписания по неделям в `localStorage` и в памяти вкладки
    - бережная политика обновления: прошлые недели не дергают API, если данные уже есть
    - предзагрузка предыдущей и следующей недели для мгновенного листания
  - На бэкенде:
    - `TTLCache` для ответов `search` и `schedule`
    - пул HTTP‑соединений `requests.Session` с retry‑политикой
    - сжатие ответов через `Flask-Compress`

- **UX‑детали**
  - Pull‑to‑refresh жест (потянуть вниз от верхушки списка пары) с кастомной анимацией кружка
  - Отдельная подсветка текущей пары, с бейджем `до конца: N мин.`
  - Особая обработка занятий по иностранному языку: карточки с несколькими преподавателями и их почтой
  - Bottom‑sheet по тапу на пару с возможностью скопировать e‑mail преподавателя

## 📌 Примеры использования

### Открытие расписания для группы

1. Откройте Mini App по ссылке `https://t.me/FinAcademiaBot` внутри Telegram.
2. Нажмите на строку "Введите группу" в шапке.
3. Начните вводить номер группы — появятся подсказки.
4. Выберите нужную группу — расписание текущей недели загрузится и будет сохранено в кэше.

### Добавление группы в избранное

1. В окне поиска найдите нужную группу.
2. Нажмите на иконку звезды справа от результата.
3. Группа появится в блоке **"Избранные"** и будет храниться в `localStorage` браузера.

### Обновление расписания

1. Откройте любой день недели.
2. Потяните содержимое списка вниз (pull‑to‑refresh), пока не появится анимированное кольцо.
3. Отпустите — фронтенд принудительно обновит данные этой недели из API через бэкенд.

## ⚙️ Конфигурация и переменные окружения

**Бэкенд (`backend/.env`, пример в `.env.example`)**

```env
PORT=8000                   # порт Flask-приложения
RUZ_BASE=https://ruz.fa.ru  # базовый URL официального API RUZ
CACHE_TTL_SECONDS=600       # TTL для in-memory кэша (секунды)
```

**Фронтенд (`frontend/.env.*`)**

- `frontend/.env.development`:

```env
VITE_API_BASE=http://localhost:8000/api
```

- `frontend/.env.production`:

```env
VITE_API_BASE=/api
```

Эти переменные определяют базовый URL для фронтенд‑запросов (`fetchJSON` в `App.jsx`).

## 🚀 Деплой

В репозитории есть файл `render.yaml` для деплоя на [Render](https://render.com):

```yaml
services:
  - type: web
    name: ruz-fa-ru-app
    env: python
    buildCommand: |
      pip install -r requirements.txt
      cd frontend && npm ci && npm run build
    startCommand: |
      gunicorn backend.app:app
```

Обратите внимание, что в продакшне:

- фронтенд собирается командой `npm run build`, результат попадает в `frontend/dist`;
- `gunicorn` запускает приложение `backend.app:app`;
- статика (`index.html`, JS и CSS) раздаётся через Flask из `frontend/dist`.

Если вы деплоите на другой хостинг, можно повторить те же шаги:

1. Собрать фронтенд.
2. Установить зависимости Python из `backend/requirements.txt`.
3. Запустить `gunicorn backend.app:app` или аналогичный WSGI‑сервер.

## ❓ FAQ

- **Q: Почему поиск групп ничего не находит?**  
  **A:** Бэкенд читает локальный файл `backend/groups_index.json`, который генерируется скриптом `build_groups_index.py`. Убедитесь, что файл существует и актуален. Для пересборки:

  ```bash
  cd backend
  python build_groups_index.py --validate
  ```

- **Q: Откуда берутся данные о расписании?**  
  **A:** Все данные приходят из официального API RUZ (`RUZ_BASE`, по умолчанию `https://ruz.fa.ru`). Бэкенд выступает только как тонкий кэширующий прокси.

- **Q: Можно ли использовать приложение без Telegram?**  
  **A:** Да. Это обычное SPA на React: достаточно открыть собранный фронтенд в браузере. В режиме браузера просто не будут активны функции Telegram WebApp (например, back‑кнопка).

- **Q: Что делать, если расписание "зависло" или кэш устарел?**  
  **A:** Для текущей недели есть pull‑to‑refresh (потянуть вниз от верха списка дней) — фронтенд принудительно обновит данные, а бэкенд перезатрёт кэш.

- **Q: Есть ли готовая конфигурация Docker / Kubernetes?**  
  **A:** _(не указано)_ — готовых манифестов нет, но приложение легко контейнеризировать, опираясь на команды сборки в этом README.
